## Basic Search

### CLI Search

```bash
npx @arabold/docs-mcp-server@latest search <library> <query>
```

<Tabs>
  <Tab title="Simple Query">
    ```bash
    npx @arabold/docs-mcp-server@latest search react "useState hook"
    ```
  </Tab>
  
  <Tab title="With Version">
    ```bash
    npx @arabold/docs-mcp-server@latest search react "useState hook" --version 18.x
    ```
  </Tab>
  
  <Tab title="All Libraries">
    ```bash
    # Omit library name to search all
    npx @arabold/docs-mcp-server@latest search "authentication"
    ```
  </Tab>
</Tabs>

## Search Ranking

Results are ranked using a hybrid approach:

<Steps>
  <Step title="Vector Search">
    Semantic similarity using embeddings (if configured)
    
    Finds conceptually related content even without exact keyword matches
  </Step>
  
  <Step title="Keyword Search">
    Full-text search using SQLite FTS5
    
    Matches exact terms and phrases
  </Step>
  
  <Step title="Reciprocal Rank Fusion">
    Combines both rankings for final results
    
    Weights: 70% vector search, 30% keyword search (configurable)
  </Step>
</Steps>

<Info>
  Vector search requires an [embedding model](/guides/embedding-models) to be configured. Without it, only keyword search is used.
</Info>

## Version Matching

Control which versions are searched:

### Version Patterns

<ParamField path="Exact Version" type="string">
  Match specific version: `18.3.1`, `3.12.0`
</ParamField>

<ParamField path="X-Range" type="string">
  Match latest within range: `18.x`, `3.x`
  
  `18.x` matches latest 18.x.x version
</ParamField>

<ParamField path="Latest" type="string">
  Match most recent version: `latest`
</ParamField>

<ParamField path="Wildcard" type="string">
  Search all versions: `*` or omit version
</ParamField>

### Examples

<Tabs>
  <Tab title="Exact Version">
    ```bash
    npx @arabold/docs-mcp-server@latest search react "hooks" --version 18.3.1
    ```
    
    Only searches React 18.3.1 documentation
  </Tab>
  
  <Tab title="X-Range">
    ```bash
    npx @arabold/docs-mcp-server@latest search python "asyncio" --version 3.x
    ```
    
    Searches latest Python 3.x documentation (e.g., 3.12 if that's the latest 3.x indexed)
  </Tab>
  
  <Tab title="Latest">
    ```bash
    npx @arabold/docs-mcp-server@latest search next.js "app router" --version latest
    ```
    
    Searches the most recent Next.js version
  </Tab>
  
  <Tab title="All Versions">
    ```bash
    npx @arabold/docs-mcp-server@latest search django "models"
    ```
    
    Searches across all indexed Django versions
  </Tab>
</Tabs>

## Query Formulation

### Best Practices

<CardGroup cols={2}>
  <Card title="Be Specific" icon="bullseye">
    "useState hook initialization" is better than "hooks"
  </Card>
  <Card title="Use Natural Language" icon="message">
    "How to handle authentication" works well with semantic search
  </Card>
  <Card title="Include Context" icon="brain">
    "React server components data fetching" vs just "data fetching"
  </Card>
  <Card title="Technical Terms" icon="code">
    Include API names, class names, or method names
  </Card>
</CardGroup>

### Query Examples

<AccordionGroup>
  <Accordion title="API Usage">
    **Good Queries:**
    - "useState hook initial state"
    - "express middleware error handling"
    - "pandas DataFrame merge join"
    
    **Avoid:**
    - "state" (too broad)
    - "how to use" (too generic)
  </Accordion>
  
  <Accordion title="Configuration">
    **Good Queries:**
    - "Next.js middleware configuration"
    - "Django database settings production"
    - "TypeScript tsconfig strict mode"
    
    **Avoid:**
    - "configuration" (too broad)
    - "settings" (too generic)
  </Accordion>
  
  <Accordion title="Troubleshooting">
    **Good Queries:**
    - "React hydration mismatch error"
    - "Python ImportError module not found"
    - "PostgreSQL connection pool exhausted"
    
    **Include:**
    - Error messages
    - Specific symptoms
    - Component or module names
  </Accordion>
</AccordionGroup>

## Search Parameters

<ParamField path="query" type="string" required>
  Search query text
</ParamField>

<ParamField path="library" type="string">
  Library name to search within (omit to search all)
</ParamField>

<ParamField path="version" type="string">
  Version pattern to match (`18.x`, `latest`, `*`)
</ParamField>

<ParamField path="limit" type="number" default="10">
  Maximum number of results to return
</ParamField>

<ParamField path="offset" type="number" default="0">
  Number of results to skip (for pagination)
</ParamField>

## Search Results

Each result includes:

<ResponseField name="title" type="string">
  Document title
</ResponseField>

<ResponseField name="content" type="string">
  Text excerpt with matched content
</ResponseField>

<ResponseField name="url" type="string">
  Source URL of the documentation
</ResponseField>

<ResponseField name="library" type="string">
  Library name
</ResponseField>

<ResponseField name="version" type="string">
  Version number
</ResponseField>

<ResponseField name="score" type="number">
  Relevance score (higher is better)
</ResponseField>

<ResponseField name="metadata" type="object">
  Additional metadata (file path, document type, etc.)
</ResponseField>

## Advanced Search

### Multi-Library Search

Search across multiple libraries:

```bash
# Search all libraries
npx @arabold/docs-mcp-server@latest search "authentication middleware"

# Results from react, express, django, etc.
```

### Version Comparison

Find differences between versions:

<Tabs>
  <Tab title="Old Version">
    ```bash
    npx @arabold/docs-mcp-server@latest search react "Suspense" --version 17.x
    ```
  </Tab>
  
  <Tab title="New Version">
    ```bash
    npx @arabold/docs-mcp-server@latest search react "Suspense" --version 18.x
    ```
  </Tab>
</Tabs>

Compare results to understand API changes.

### Pagination

Retrieve more results:

```bash
# First 10 results
npx @arabold/docs-mcp-server@latest search python "asyncio" --limit 10

# Next 10 results
npx @arabold/docs-mcp-server@latest search python "asyncio" --limit 10 --offset 10
```

### Search Configuration

Tune search behavior:

<ParamField path="search.vectorWeight" type="number" default="0.7">
  Weight for vector search in RRF (0.0 to 1.0)
</ParamField>

<ParamField path="search.overfetchFactor" type="number" default="3">
  Retrieve 3x more candidates before final ranking
</ParamField>

```bash
# Prefer keyword search
DOCS_MCP_SEARCH_VECTOR_WEIGHT=0.3 npx @arabold/docs-mcp-server@latest search react "hooks"

# Prefer semantic search
DOCS_MCP_SEARCH_VECTOR_WEIGHT=0.9 npx @arabold/docs-mcp-server@latest search react "hooks"
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="No Results Found">
    **Problem:** Search returns empty results
    
    **Solutions:**
    - Verify library is indexed: `npx @arabold/docs-mcp-server@latest list`
    - Check version pattern matches indexed versions
    - Try broader query terms
    - Search all libraries (omit `--library`)
    - Check indexing completed successfully
  </Accordion>
  
  <Accordion title="Irrelevant Results">
    **Problem:** Results don't match query intent
    
    **Solutions:**
    - Make query more specific
    - Include more context terms
    - Enable embedding model for semantic search
    - Adjust `search.vectorWeight` configuration
    - Use exact phrases in quotes
  </Accordion>
  
  <Accordion title="Version Not Found">
    **Problem:** Error: "No versions found matching pattern"
    
    **Solutions:**
    - List available versions: `npx @arabold/docs-mcp-server@latest list`
    - Use `--version *` to search all versions
    - Check version pattern syntax (`18.x` not `18.X`)
    - Verify version was indexed successfully
  </Accordion>
  
  <Accordion title="Slow Search">
    **Problem:** Search takes too long
    
    **Solutions:**
    - Reduce `--limit` (default: 10)
    - Search specific library instead of all
    - Use version pattern to narrow scope
    - Check database size and vacuum if needed
  </Accordion>
</AccordionGroup>

## Search from AI Assistant

Your AI assistant can search documentation automatically:

### Example Prompts

<AccordionGroup>
  <Accordion title="Direct Search">
    **Prompt:**
    > "Search the React documentation for information about the useEffect hook cleanup function"
    
    AI uses `search_docs` tool with appropriate query
  </Accordion>
  
  <Accordion title="Version-Specific">
    **Prompt:**
    > "What's new in React 18 regarding Suspense? Check the React 18.x documentation."
    
    AI searches with `version: "18.x"` parameter
  </Accordion>
  
  <Accordion title="Multi-Step">
    **Prompt:**
    > "How do I implement authentication in Express? First list what Express versions we have, then search the latest."
    
    AI uses `list_libraries` then `search_docs`
  </Accordion>
  
  <Accordion title="Comparison">
    **Prompt:**
    > "Compare how routing works in Next.js 13 versus Next.js 14"
    
    AI performs multiple searches with different version patterns
  </Accordion>
</AccordionGroup>

<Tip>
  Train your AI by showing it example searches in the Web UI first, then use similar prompts.
